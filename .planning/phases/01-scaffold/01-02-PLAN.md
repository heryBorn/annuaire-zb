  <files>
    annuaire-zb-react/tailwind.config.js

  <action>
    From inside annuaire-zb-react/:

    ```bash
    cd D:/Workspace/Php/www/annuaire-zb/annuaire-zb-react
    npm install -D tailwindcss postcss autoprefixer
    npx tailwindcss init -p
    ```

    The `init -p` flag generates both tailwind.config.js AND postcss.config.js in one step — do NOT manually create postcss.config.js (see Pitfall 3 in RESEARCH.md).

    Then replace the generated tailwind.config.js with the full project config:

    ```js
    /** @type {import('tailwindcss').Config} */
    module.exports = {
      content: ['./src/**/*.{js,jsx,ts,tsx}', './public/index.html'],
      theme: {
        extend: {
          colors: {
            soil:       '#2C1A0E',
            terracotta: '#C1440E',
            sand:       '#F5E6C8',
            wheat:      '#E8C97A',
            sage:       '#6B8F71',
            cream:      '#FAF5EC',
            ink:        '#1A1108',
            muted:      '#8A7A6A',
          },
          fontFamily: {
            sans:  ['"DM Sans"', 'sans-serif'],
            serif: ['"Playfair Display"', 'serif'],
          },
        },
      },
      plugins: [],
    }
    ```

    Critical: the `content` array MUST include `./src/**/*.{js,jsx,ts,tsx}` with explicit jsx extension — omitting it causes Tailwind to strip all custom color classes in the production build (Pitfall 2 in RESEARCH.md).
  </action>
  <verify>
    <automated>cd D:/Workspace/Php/www/annuaire-zb/annuaire-zb-react && node -e "const c=require('./tailwind.config.js'); const colors=c.theme.extend.colors; ['soil','terracotta','sand','wheat','sage','cream','ink','muted'].forEach(k=>{if(!colors[k]) process.exit(1)}); if(!c.content.some(p=>p.includes('jsx'))) process.exit(1); console.log('Tailwind config OK')"</automated>
  </verify>
  <done>tailwind.config.js has all 8 color tokens and content glob includes jsx; postcss.config.js exists</done>
</task>
