```jsx
import AvailabilityBadge from './AvailabilityBadge';

function MemberCard({ member: m }) {
  const initials = ((m.prenom || '?')[0] + (m.nom || '?')[0]).toUpperCase();

  return (
    <div className="
      rounded-xl overflow-hidden shadow-md bg-white cursor-pointer
      transition-all duration-200 ease-out
      hover:shadow-xl hover:scale-[1.02]
    ">
      {/* Square photo — aspect-square ensures consistent card height regardless of image dimensions */}
      <div className="w-full aspect-square overflow-hidden bg-sand flex items-center justify-center">
        {m.photo_url
          ? <img src={m.photo_url} alt={`${m.prenom} ${m.nom}`} className="w-full h-full object-cover" />
          : <span className="font-serif text-3xl text-muted">{initials}</span>
        }
      </div>

      {/* Info stacked below photo: name → title → company → badge */}
      <div className="p-4">
        <p className="font-serif font-semibold text-ink leading-tight">
          {m.prenom} {m.nom}
        </p>
        <p className="font-sans text-sm text-muted mt-0.5 truncate">
          {m.metier || '\u00A0'}
        </p>
        <p className="font-sans text-sm text-muted truncate">
          {m.entreprise || '\u00A0'}
        </p>
        <AvailabilityBadge disponibilite={m.disponibilite} />
      </div>
    </div>
  );
}

export default MemberCard;
```

Critical rules:
- Locked decision: photo is a square filling the card width — enforce with `aspect-square` + `object-cover`. Do NOT use a fixed height like `h-48`.
- Locked decision: hover effect is shadow increase + `scale(1.02)` via CSS only — no onClick state or JS transform.
- Locked decision: info stacked as name → title → company → badge in that exact order.
- Use `'\u00A0'` (non-breaking space) as fallback for empty metier/entreprise rows — keeps card height consistent when fields are empty.
- initials fallback mirrors existing index.html `initials()` function pattern.
- `truncate` on title/company prevents overflow on long values.

```jsx
        {/* title row */}

        {/* company row */}
        <div className="h-3 bg-sand rounded w-2/3" />
        {/* badge row */}

      </div>
    </div>
  );
}

export default SkeletonCard;
```

Critical rules:
- `aspect-square` on the image placeholder is mandatory — matches MemberCard and prevents layout shift when data loads.
- `animate-pulse` goes on the outer wrapper, not individual rows — a single pulse animation is less visually noisy.
- Use `bg-sand` (project color token) for placeholder fills — matches the project's earthy palette.
- The 4 text rows (name, title, company, badge) must mirror MemberCard's info section — same count, similar proportions.
- This component takes no props — it is purely a layout placeholder.
